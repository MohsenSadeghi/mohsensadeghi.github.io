About IBMD

The IBMD (In-Browser Molecular Dynamics) is a web application that performs Molecular Dynamics (MD) simulations and shows the results in real-time (in the computational sense of the word), all inside a standard web browser. The details of this particular simulation case are given in the following sections.

The IBMD lets you see physical and thermodynamic properties of the simulated system as the simulation progresses. The web application relies on Javascript for doing the necessary computations and employs HTML5 capabilities for presentation. The processes are all client-side and they use your computer's, tablet's, or phone's CPU. The simulation has no server-side computational cost. So, feel free to continue the simulation for as long as you like!

Case 4: Argon-Neon.2D.LJ.NVT


Simulation box

The simulation box consists of 225 atoms, 30% argon and 70% neon. The mixture is placed in a 2-dimensional square with sides 10 nm in length. Atoms are initially positioned on an ordered lattice and given the same speed with random moving directions. Argon and neon atoms are initially separated. Periodic boundary conditions are applied in the X and Y directions. The minimum image convention is used to handle the periodic images in calculating interaction energies and forces.

Statistical ensemble

The simulation is carried out in the canonical or NVT ensemble, which means that the number of particles (N), the volume of the system (V), and its temperature (T) are constant.

Interactions

Lennard-Jones 6-12 potential function is used to model the pair-wise interactions of argon and neon atoms. The parameters of the potential are taken from information provided by SklogWiki pages on argon and neon.

Lorentz-Berthelot mixing rule is used for crossed interactions. You may visit this SklogWiki page on different mixing rules. No cut-off other than what is forced by the minimum image convention is used.

Time integration

Equations of motion are integrated using the velocity Verlet algorithm. You can learn more about this algorithm from this Wikipedia page on Verlet integration.

A timestep of 10 fs is used for time integration.

Thermostat

In order to maintain constant temperature in the simulation, the Berendsen thermostat is used. The Berendsen thermostat acts through scaling velocity vectors based on the ratio of current temperature to the setpoint temperature.

Output

To the right of the page, you see an animated output showing the positions of particles in the simulation box. The particles are illustrated by circles with radii equal to the actual van der Waals radii of Argon atoms. A small scale-bar at the lower left corner of the picture shows the scale at which you are seeing the simulation box.

To the left, there is a table displaying thermodynamic properties of the system. Using Real-time Plot option, you can plot graphs of these properties versus time and see them fluctuate as the simulation progresses or as you use the controls.

Controls

The simulation starts by pressing the Start button and stops by pressing Stop. You can resume a stopped simulation by pressing the Start button agian. You can also restart a simulation, whether running or stopped, to the initial state by pressing the Reset button.

You can change setpoint temperature using T ▲ and T ▼ buttons. When the setpoint temperure changes, the Berendsen thermostat starts adjusting the system temperature to match the setpoint value. But this process takes time and you have to wait briefly for the system temperature to reach the designated value. You can check this by plotting temperature versus time and monitoring the fluctuations.
